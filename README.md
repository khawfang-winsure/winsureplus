# WIN SURE PLUS — ระบบติดตามลูกค้าผ่อนเครื่อง

เว็บกลางสำหรับธุรกิจผ่อน iPhone ผ่านร้านค้าพาร์ทเนอร์ — กรอกข้อมูล **ครั้งเดียว** แล้วได้ครบ:
เก็บข้อมูลลูกค้า/ร้านค้า + สร้างข้อความสรุปยอดโอน + สร้างข้อความอีเมล + ติดตามสถานะการชำระ

> มาแทน Google Sheets หลายชีต ที่ไม่ real-time และตรวจสอบยาก

---

## สถานะตอนนี้ (Phase 0–1 บางส่วน)

✅ โครงเว็บ + เมนูซ้าย ตามแบบที่ออกแบบไว้
✅ หน้า **เพิ่มข้อมูลสัญญา** — กรอกครั้งเดียว คำนวณยอดสด + สร้างข้อความสรุปยอด/อีเมล พร้อมปุ่มคัดลอก
✅ หน้า **ลูกค้าทั้งหมด** (ตาราง) และหน้า **ตั้งค่า** (ดูร้านค้า/ตัวเลือก)
🔜 ยังเป็น **ข้อมูลตัวอย่าง (mock)** — ยังไม่เชื่อมฐานข้อมูลจริง

ดูแผนเต็มได้ที่ส่วน "ขั้นต่อไป" ด้านล่าง

---

## วิธีเปิดรันในเครื่อง

ต้องมี [Node.js](https://nodejs.org/) (เวอร์ชัน 18 ขึ้นไป) ก่อน

```bash
npm install      # ติดตั้งครั้งแรกครั้งเดียว
npm run dev      # เปิดเว็บที่ http://localhost:5173
```

ก่อน deploy ขึ้นจริง ให้รัน `npm run build` เช็คว่าไม่มี error

## โครงไฟล์

```
src/
  components/    ส่วน UI ที่ใช้ซ้ำ (เมนู, การ์ด, ปุ่ม, กล่องคัดลอก)
  pages/         แต่ละหน้าในเมนู (เพิ่มสัญญา, ลูกค้าทั้งหมด, ...)
  lib/
    types.ts     พิมพ์เขียวข้อมูล (จะ map กับตาราง Supabase)
    calc.ts      สูตรคำนวณ (ยอดโอน, วันครบกำหนด, ค่าปรับ)
    messages.ts  ตัวสร้างข้อความสรุปยอด/อีเมล
    format.ts    จัดรูปแบบตัวเลข/วันที่/ช่วงอายุ
    mockData.ts  ข้อมูลตัวอย่าง (จะแทนด้วยข้อมูลจริงจาก Supabase)
```

## ขั้นต่อไป (ตามแผนเป็นเฟส)

1. **เชื่อม Supabase** — สร้างตาราง + ใส่กุญแจในไฟล์ `.env` (ดู `.env.example`)
2. **บันทึกข้อมูลจริง** — แทน mockData ด้วยข้อมูลจาก Supabase
3. **ตารางงวดผ่อน + สถานะล่าช้าอัตโนมัติ** (กลุ่ม 1-10 / 11-30 / ...)
4. **ระบบรันวันล่าช้า + ค่าปรับ อัตโนมัติรายวัน** (pg_cron)
5. **คืนเครื่อง 3 กรณี + ค่าซ่อม**
6. **สิทธิ์ admin/พนักงาน + ย้ายข้อมูลเก่า**

> กฎธุรกิจสำคัญ (สูตรยอดโอน, วันครบกำหนด clamp สิ้นเดือน, ค่าปรับ 700฿/งวด ฯลฯ)
> เขียนไว้เป็นคอมเมนต์ในไฟล์ `src/lib/calc.ts` แล้ว
