# ดีไซน์: รายงานวัดผลร้านค้า (Shop Report)

วันที่: 2026-06-08

## เป้าหมาย
ให้แอดมินดูผลของร้านค้าพาร์ทเนอร์แต่ละร้าน — ทั้งปริมาณงานและคุณภาพลูกค้า —
แล้วสรุปเป็น "เกรด" เพื่อตัดสินใจเรื่องโปรโมชั่น/ความเสี่ยง

## กฎสำคัญ (ยืนยันกับ Pete)
- **"หนี้เสีย" มี 2 เกณฑ์คนละมุม:**
  - ฝั่งลูกค้า = 60 วัน (ใช้ในการติดตามลูกค้า — ไม่ใช้ในรายงานนี้)
  - **ฝั่งร้านค้า (รายงานนี้) = ล่าช้า 31 วันขึ้นไป** ถือเป็น "ลูกค้าเสี่ยง" ที่ดึงเกรดร้านให้ตก
- ในรายงานนี้เลี่ยงคำว่า "หนี้เสีย" → ใช้ "ลูกค้าเสี่ยง (ล่าช้า 31 วัน+)" แทน เพื่อกันสับสน

## ข้อมูลต่อร้าน
- จำนวนสัญญา (ปริมาณ)
- ยอดขายรวม = ผลรวมราคาเครื่องของทุกสัญญา
- ลูกค้าดี = สัญญาที่ไม่เข้าเกณฑ์เสี่ยง
- ลูกค้าเสี่ยง = สัญญาที่ status=active และล่าช้า 31 วันขึ้นไป (bucket: 31-60/61-90/91-120/120+)
- % เสี่ยง = ลูกค้าเสี่ยง / จำนวนสัญญา
- เกรด

## เกณฑ์เกรด
- A (เขียว): เสี่ยง < 5%
- B (ฟ้า): 5–15%
- C (ส้ม): 15–30%
- D (แดง): > 30%

## หน้าจอ
- เมนูใหม่ "รายงานร้านค้า"
- สรุปด้านบน: จำนวนร้านทั้งหมด / เกรด A กี่ร้าน / เกรด D (ต้องระวัง) กี่ร้าน
- ตารางร้านค้า เรียงจากเกรดดีสุด: ร้าน | สัญญา | ยอดขาย | ดี | เสี่ยง | %เสี่ยง | เกรด(ป้ายสี)
- คลิกร้าน → (อนาคต) ดูรายชื่อลูกค้าของร้านนั้น

## วิธีทำ (สถาปัตยกรรม)
- **ไม่ต้องเพิ่ม SQL migration** — คำนวณฝั่ง frontend จากข้อมูลที่มี:
  `getShops()` + `getContracts()` + `getAllStatuses()` (v_contract_status มี bucket อยู่แล้ว)
- `src/lib/report.ts`: ฟังก์ชันบริสุทธิ์ `buildShopReport(shops, contracts, statuses) -> ShopReportRow[]`
- `src/pages/ShopReport.tsx`: โหลดข้อมูล → เรียก buildShopReport → แสดงตาราง
- เพิ่มเมนู + route

## ขอบเขต (YAGNI)
- เฟสนี้: ตารางเกรด + สรุป เท่านั้น
- ยังไม่ทำ: กราฟ, ช่วงเวลา (filter by date), drill-down รายชื่อลูกค้าต่อร้าน — ไว้เฟสถัดไปถ้าต้องการ
