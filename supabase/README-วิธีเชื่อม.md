# วิธีเชื่อม Supabase (สำหรับเตย — ทำตามทีละขั้น)

> ทำครั้งเดียว ใช้เวลาประมาณ 15 นาที ไม่ต้องเขียนโค้ดเอง แค่ก๊อป-วาง-กด

---

## ขั้นที่ 1: สร้างโปรเจกต์ใน Supabase

1. เข้า [supabase.com](https://supabase.com) → ล็อกอินบัญชีของเตย
2. กดปุ่ม **New Project** (สีเขียว)
3. ตั้งชื่อ เช่น `winsureplus` → ตั้ง **Database Password** (จดไว้ดีๆ) → เลือก Region **Southeast Asia (Singapore)**
4. กด **Create new project** แล้วรอสักครู่ (~2 นาที) ให้ระบบสร้างเสร็จ

---

## ขั้นที่ 2: สร้างตาราง (ก๊อป SQL ไปวาง)

1. ในเมนูซ้าย กดไอคอน **SQL Editor** (รูป `</>`)
2. กด **+ New query**
3. เปิดไฟล์ [`migrations/0001_init.sql`](migrations/0001_init.sql) → ก๊อปทั้งหมด → วางในช่อง
4. กดปุ่ม **Run** (หรือ Ctrl+Enter) → ควรขึ้น **Success** สีเขียว
5. (ไม่บังคับ) ทำซ้ำกับไฟล์ [`seed.sql`](seed.sql) เพื่อใส่ร้านค้า/ตัวเลือกตัวอย่าง

> ✅ SQL ชุดนี้ทดสอบแล้วว่ารันผ่าน — ถ้าขึ้น error สีแดง ส่งข้อความ error มาให้ดูได้เลย

---

## ขั้นที่ 3: เอา "กุญแจ" มาใส่ในเว็บ

1. เมนูซ้าย → **Project Settings** (รูปเฟือง) → **API**
2. จะเห็น 2 ค่า:
   - **Project URL** (เช่น `https://xxxx.supabase.co`)
   - **anon public** key (ข้อความยาวๆ)
3. ในโฟลเดอร์โปรเจกต์ คัดลอกไฟล์ `.env.example` เป็นชื่อ `.env` แล้วใส่ค่า:
   ```
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=ค่า anon public ที่ก๊อปมา
   ```
4. บอกครีม (หรือพี่พิธ) ว่าใส่เสร็จแล้ว → ครีมจะต่อโค้ดให้เว็บดึงข้อมูลจริงแทนข้อมูลตัวอย่าง

---

## ขั้นที่ 4: สร้างผู้ใช้ + ตั้งเป็นแอดมิน

1. เมนูซ้าย → **Authentication** → **Users** → **Add user** → ใส่อีเมล/รหัสผ่านของเตย
2. กลับไป **SQL Editor** รันคำสั่งนี้ (เปลี่ยนอีเมลเป็นของเตย) เพื่อตั้งเป็นแอดมิน:
   ```sql
   insert into profiles (id, role, full_name)
   select id, 'admin', email from auth.users where email = 'อีเมลของเตย'
   on conflict (id) do update set role = 'admin';
   ```
   > คำสั่งนี้ใช้ได้ทั้งกรณีมี profile อยู่แล้วและยังไม่มี (เช่น สร้าง user ก่อนรัน SQL)

เสร็จแล้ว! เว็บจะเชื่อมกับฐานข้อมูลจริง บันทึก/ดึงข้อมูลได้ครบ 🎉

---

### หมายเหตุ
- กุญแจ `anon key` ใส่ในเว็บได้ปลอดภัย (ออกแบบมาให้เปิดเผยฝั่งหน้าเว็บได้) — ความปลอดภัยจริงคุมด้วยกฎ RLS ที่อยู่ใน `0001_init.sql` แล้ว
- ไฟล์ `.env` จะ **ไม่** ขึ้น GitHub (กันกุญแจหลุด) — ตั้งค่าไว้ใน `.gitignore` แล้ว
